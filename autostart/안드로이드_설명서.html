1

개요 — 안드로이드 전자칠판용 알림이란?

교무실에서 학생을 호출하면 해당 교실의 안드로이드 전자칠판에 큰 알림과 호출음·음성 안내가 즉시 표시됩니다. APK를 설치하면 전자칠판이 꺼진 상태에서도 화면이 자동으로 켜지며, 재부팅 후에도 Chrome이 자동 실행됩니다.

📺
화면 자동 활성화
평소에는 검은 화면 대기. 호출 시 화면이 켜지며 알림 표시
🔊
소리 + 음성 안내
호출음과 TTS 음성으로 학생 이름·교실 정보 즉시 안내
🔄
재부팅 후 자동 실행
전자칠판 재부팅 시 APK의 BootReceiver가 Chrome을 자동 실행
🛡️
강제 종료 자동 복구
Chrome이 종료되면 WatchdogService가 15초 내에 자동 재실행
2

요구사항

📱
안드로이드 기반 전자칠판 Chrome 브라우저가 설치된 안드로이드 스마트칠판. 개발자 옵션 및 USB 디버깅 활성화 필요.
🌐
Google Chrome (전자칠판 내) Chrome이 전자칠판에 설치되어 있어야 합니다. 알림 시스템은 Chrome을 통해 동작합니다.
📶
Wi-Fi 인터넷 연결 Firebase 실시간 통신을 사용합니다. 전자칠판이 교내 Wi-Fi에 연결되어 있어야 합니다.
🔌
USB 케이블 + ADB (PC) APK 설치 시 USB 케이블로 PC와 전자칠판을 연결해야 합니다. ADB는 platform-tools에 포함되어 있으며 설치 bat가 자동으로 사용합니다.
🏗️
Android Studio (APK 빌드 시, 1회만) APK를 직접 빌드할 경우 필요합니다. 빌드는 PC에서 1회만 수행하며, 이후 같은 APK를 여러 교실에 반복 설치할 수 있습니다.
3

사전 준비 — 전자칠판 설정

APK 설치 전, 전자칠판에서 개발자 옵션과 USB 디버깅을 한 번만 활성화해야 합니다.

1

개발자 옵션 활성화

전자칠판에서 설정 → 디바이스 정보 → 빌드 번호를 찾아 7번 연속으로 탭합니다.

"개발자 옵션이 활성화되었습니다" 메시지가 나타나면 성공입니다.

💡
빌드 번호 위치가 다른 경우

전자칠판 제조사마다 위치가 다를 수 있습니다. 설정에서 "빌드 번호"를 검색하거나, 설정 → 시스템 → 디바이스 정보 경로를 시도해 보세요.

2

USB 디버깅 활성화

설정 → 개발자 옵션으로 이동한 뒤 USB 디버깅 항목을 켭니다.

PC와 USB로 연결하면 "USB 디버깅을 허용하시겠습니까?" 팝업이 나타납니다. "이 컴퓨터에서 항상 허용"에 체크 후 확인을 누릅니다.

⚠️
팝업이 나타나지 않을 경우

USB 케이블을 뽑았다가 다시 연결하거나, PC에서 ADB 드라이버가 제대로 설치되어 있는지 확인하세요.

⬇

APK 다운로드 — USB 없이 바로 설치

🎉
Android Studio 불필요 — 전자칠판에서 바로 다운로드

아래 버튼을 전자칠판 Chrome에서 탭하면 APK를 직접 다운로드할 수 있습니다. USB 케이블이나 PC 없이 설치 가능합니다.

📦
교실알림 자동실행 APK
최신 버전 · GitHub Releases
⬇ APK 다운로드
파일명: app-debug.apk
1

전자칠판에서 이 설명서 페이지 열기

전자칠판의 Chrome에서 이 페이지를 열고 위 APK 다운로드 버튼을 탭합니다.

2

출처를 알 수 없는 앱 허용

다운로드 후 설치 시 "이 출처의 앱 설치 허용" 팝업이 나타납니다. 허용을 탭합니다.

💡
팝업이 나타나지 않는 경우

설정 → 보안 → 출처를 알 수 없는 앱 (또는 앱 설치 허용)에서 Chrome을 허용하세요. 전자칠판 제조사마다 위치가 다를 수 있습니다.

3

설치 완료 → 5번 설치 방법으로 이동

APK 설치 후 아래 5. 설치 방법 섹션의 3단계(전자칠판에서 앱 설정)부터 진행하세요.

4

APK 빌드 — Android Studio (1회)

ℹ️
빌드는 1회만 필요합니다

APK를 한 번 빌드해두면 같은 파일을 여러 교실 전자칠판에 반복 설치할 수 있습니다. 이미 APK 파일이 있다면 이 단계를 건너뛰세요.

1

Android Studio 설치

PC에 Android Studio가 없다면 developer.android.com/studio 에서 무료로 다운로드하여 설치합니다.

2

프로젝트 열기

Android Studio를 실행한 뒤 Open을 클릭하고, 아래 폴더를 선택합니다.

autostart/android-autostart

Gradle 동기화가 자동으로 시작됩니다. 완료될 때까지 기다립니다.

3

APK 빌드

메뉴에서 Build → Build Bundle(s) / APK(s) → Build APK(s) 를 클릭합니다.

빌드가 완료되면 우측 하단에 알림이 나타납니다. "locate" 링크를 클릭하면 APK 파일 위치가 열립니다.

// APK 생성 위치 (예시)
android-autostart/app/build/outputs/apk/debug/app-debug.apk
💡 빌드 없이 ADB만으로 설치하는 방법 (자동 실행 기능 제한)

APK 없이 안드로이드_설치.bat를 실행하면 반 번호를 입력하고 Chrome을 바로 실행할 수 있습니다.

단, 이 방법은 재부팅 후 자동 실행 기능이 없습니다. 전자칠판을 재시작하면 Chrome을 수동으로 열어야 합니다. 완전한 자동화를 위해서는 APK 설치를 권장합니다.

5

설치 방법 — 교실별 설치

1

USB 케이블로 PC와 전자칠판 연결

PC와 전자칠판을 USB 케이블로 연결합니다. 전자칠판 화면에 "USB 디버깅을 허용하시겠습니까?" 팝업이 나타나면 확인을 탭합니다.

2

안드로이드_APK_설치.bat 실행

📄 안드로이드_APK_설치.bat 파일을 더블클릭합니다.

bat 파일이 자동으로 다음 작업을 수행합니다:

✔APK 파일 자동 탐색 (android-autostart 폴더 내)
✔ADB를 통해 전자칠판에 APK 설치
✔배터리 최적화 제외 권한 자동 설정
3

전자칠판에서 앱 설정

전자칠판 앱 서랍에서 "교실알림 자동실행" 앱을 실행합니다.

학년과 반을 선택한 뒤 [저장 + 실행] 버튼을 탭합니다.

앱이 Chrome을 자동으로 열고 해당 교실의 알림 페이지로 이동합니다.

4

Chrome에서 오디오 활성화

Chrome이 열리면 화면에 "화면을 터치하여 시작" 안내가 표시됩니다.

전자칠판 화면을 한 번 탭하여 오디오를 활성화합니다.

🔊
이 단계가 중요합니다

브라우저의 자동 재생 정책으로 인해 반드시 화면을 한 번 터치해야 호출 소리가 정상 재생됩니다. 터치하지 않으면 알림은 표시되지만 소리가 나지 않습니다.

5

설치 완료 — 화면이 검게 변하며 대기

오디오 활성화 후 약 2초 뒤 화면이 검은색으로 전환됩니다. 이것이 정상 대기 상태입니다.

검은 화면 상태에서 교무실로부터 호출이 오면 화면이 켜지며 알림이 표시됩니다.

6

동작 방식

⬛
평상시 — 검은 화면 대기 Chrome이 실행 중이지만 화면은 절전 모드(검은 화면)로 유지됩니다. 전력 소모를 줄이고 수업에 방해되지 않습니다.
🔔
호출 수신 — 화면 자동 활성화 + 알림 표시 교무실에서 호출하면 화면이 켜지며 중앙에 큰 알림이 표시됩니다. 호출음과 TTS 음성이 함께 재생됩니다.
⬛
호출 종료 — 다시 검은 화면으로 복귀 알림 확인 후 또는 일정 시간이 지나면 화면이 다시 검은 화면으로 돌아갑니다.
👆
검은 화면 터치 — 10초간 상태 표시 검은 화면을 터치하면 현재 연결 상태와 교실 정보가 10초간 표시된 후 다시 검은 화면으로 돌아갑니다.
7

자동 실행 기능

🔁
재부팅 후 자동 실행 (BootReceiver) 전자칠판을 재시작하면 APK의 BootReceiver가 BOOT_COMPLETED 이벤트를 감지하여 15초 후 Chrome을 자동으로 실행합니다.
🛡️
강제 종료 자동 복구 (WatchdogService) 포그라운드 서비스로 상주하며 Chrome이 종료되었는지 15초마다 확인합니다. Chrome이 없으면 즉시 재실행합니다.
🔋
배터리 최적화 제외 APK 설치 시 배터리 최적화에서 자동으로 제외됩니다. 절전 모드에서도 서비스가 종료되지 않고 정상 동작합니다.
📱
MainActivity — 반 정보 설정 앱을 실행하면 학년·반을 선택할 수 있는 간단한 화면이 나타납니다. 설정값은 SharedPreferences에 저장되어 재부팅 후에도 유지됩니다.
⚡
START_STICKY 서비스 WatchdogService는 START_STICKY로 설정되어 있어 시스템이 서비스를 강제 종료해도 자동으로 재시작됩니다.
🎯
교실 정보 유지 저장된 학년·반 정보를 기반으로 재부팅 후에도 정확한 교실 알림 페이지가 자동으로 열립니다.
8

제거 방법

방법 1

bat 파일로 제거 (권장)

PC와 전자칠판을 USB로 연결한 상태에서 🗑️ 안드로이드_제거.bat를 더블클릭합니다.

APK가 자동으로 삭제됩니다.

방법 2

ADB 명령어로 직접 제거

PC에서 터미널(명령 프롬프트)을 열고 아래 명령어를 실행합니다:

adb uninstall com.classroom.autostart
ℹ️
재설치 시

문제가 발생했을 경우 제거 후 다시 설치하면 대부분 해결됩니다. 제거 → APK 설치 bat → 앱에서 반 선택 → Chrome 터치 순서로 진행하세요.

9

문제 해결

Q
소리가 나지 않아요
Chrome에서 "화면을 터치하여 시작"을 아직 탭하지 않았습니다. 화면을 한 번 터치해 오디오를 활성화하세요.
전자칠판 볼륨이 음소거되어 있는지 확인하세요. 볼륨 버튼으로 소리를 높여 보세요.
Chrome 주소창 옆 🔊 아이콘이 음소거 상태이면 탭하여 해제하세요.
Q
ADB 연결이 안 돼요 / 디바이스를 찾을 수 없어요
전자칠판에서 USB 디버깅이 켜져 있는지 확인하세요. (설정 → 개발자 옵션)
USB 케이블을 뽑았다가 다시 연결하고, 전자칠판 화면에 나타나는 허용 팝업에서 확인을 탭하세요.
다른 USB 포트 또는 다른 케이블을 사용해 보세요. 충전 전용 케이블은 데이터 전송이 안 됩니다.
전자칠판 USB 연결 모드가 파일 전송(MTP) 또는 PTP로 설정되어 있는지 확인하세요.
Q
화면이 검게 유지되며 알림이 표시되지 않아요
전자칠판이 Wi-Fi에 연결되어 있는지 확인하세요.
검은 화면을 터치하여 연결 상태를 확인합니다. "연결됨"이 표시되지 않으면 Chrome을 새로 고침하세요.
Chrome을 닫고 "교실알림 자동실행" 앱을 다시 실행하여 [저장 + 실행]을 탭하세요.
Q
재부팅 후 Chrome이 자동으로 열리지 않아요
APK가 정상적으로 설치되었는지 확인하세요. 앱 서랍에서 "교실알림 자동실행" 앱을 찾을 수 있어야 합니다.
재부팅 후 최대 15초를 기다려야 합니다. 즉시 켜지지 않는 것은 정상입니다.
배터리 최적화에서 앱이 제외되어 있는지 확인하세요. 설정 → 배터리 → 배터리 사용량 → 교실알림 자동실행 → 최적화 안 함.
문제가 지속되면 APK를 제거하고 다시 설치하세요.
Q
설치 bat 실행 시 ADB를 찾을 수 없다고 나와요
bat 파일은 android-autostart 폴더 내 platform-tools를 자동으로 탐색합니다. platform-tools 폴더가 올바른 위치에 있는지 확인하세요.
없다면 developer.android.com 에서 Android SDK Platform-Tools를 다운로드하여 android-autostart 폴더 안에 압축 해제하세요.
Q
연결 끊김 또는 오프라인 표시가 나와요
전자칠판의 Wi-Fi 연결 상태를 확인하세요. 교내 네트워크에 연결되어 있어야 합니다.
인터넷이 복구되면 자동으로 재연결됩니다. 수동 새로고침은 필요하지 않습니다.
방화벽이나 네트워크 정책으로 Firebase 접속이 차단될 수 있습니다. 네트워크 관리자에게 문의하세요.

주의사항

🏷️
반 정보는 앱에서 설정

APK 설치 후 반드시 "교실알림 자동실행" 앱을 열어 학년·반을 선택하고 저장해야 합니다. 미설정 시 잘못된 교실의 알림을 수신할 수 있습니다.

👆
설치 후 화면 터치 필수

Chrome이 열린 후 반드시 화면을 한 번 터치해 오디오를 활성화하세요. 이 과정을 생략하면 소리가 나지 않습니다.

⏱️
재부팅 후 15초 대기

재부팅 후 Chrome이 자동 실행되기까지 최대 15초가 걸립니다. 화면이 바로 켜지지 않아도 정상입니다.

🔌
USB 케이블은 데이터 지원 필수

충전 전용 케이블은 ADB 통신이 되지 않습니다. 반드시 데이터 전송을 지원하는 USB 케이블을 사용하세요.

📶
Wi-Fi 상시 연결 유지

알림 시스템은 인터넷을 통해 동작합니다. 전자칠판의 Wi-Fi 연결이 끊기면 알림을 수신할 수 없습니다.

🔧
문제 발생 시 재설치

알 수 없는 문제가 발생하면 제거 bat 실행 → APK 재설치 → 앱에서 반 설정 → Chrome 터치 순으로 재설치하면 대부분 해결됩니다.

🔴
주의해야 할 사항
  • APK 설치 후 반 설정 없이 방치하지 마세요. 다른 교실 알림이 수신될 수 있습니다.
  • 개발자 옵션이 활성화된 상태로 교내 네트워크에 연결되므로, 설치 완료 후 USB 디버깅은 유지하거나 비활성화해도 무관합니다.
  • 배터리 최적화를 다시 활성화하면 WatchdogService가 절전 모드에서 종료될 수 있습니다.
