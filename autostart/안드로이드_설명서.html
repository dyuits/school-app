1. 개요

교실 알림 시스템은 교무실에서 학생을 호출할 때 해당 교실의 전자칠판에 실시간으로 알림을 표시하는 스마트 교실 관리 솔루션입니다.
Firebase 기반 실시간 푸시와 TTS 음성 안내로 수업 중 불필요한 방해를 최소화합니다.

🏫 교무실
학생 호출 입력
➜
☁️ Firebase
실시간 전송
➜
📺 전자칠판
알림 + 음성 출력
➜
👨‍🎓 학생
호출 인지
ℹ️
이 설명서는 안드로이드(Android) OS가 탑재된 전자칠판을 기준으로 작성되었습니다. Windows PC 전자칠판 설명서와 디자인 색상(초록/청록)으로 구분됩니다.

3. 사전 준비 (전자칠판에서)

📌
아래 단계는 전자칠판 화면에서 직접 진행합니다. USB 케이블 연결 전에 USB 디버깅을 먼저 활성화해야 합니다.
1

🔧 개발자 모드 활성화

전자칠판의 설정(톱니바퀴) 앱을 열고 아래 경로로 이동합니다.

⚙️ 설정› 태블릿 정보› 빌드 번호

빌드 번호 항목을 7번 연속으로 터치합니다.
"개발자가 되었습니다" 또는 "개발자 옵션이 활성화되었습니다" 메시지가 나타납니다.

📱

[ 태블릿 정보 → 빌드 번호 7번 연속 터치 ]

메시지: "개발자가 되었습니다" ✓

2

🐛 USB 디버깅 켜기

개발자 옵션 메뉴에서 USB 디버깅을 활성화합니다.

⚙️ 설정› 개발자 옵션› USB 디버깅 🔵 ON

경고 다이얼로그가 표시되면 [확인]을 터치합니다.

💡
일부 기기에서는 "개발자 옵션"이 [일반] 또는 [시스템] 메뉴 하위에 있을 수 있습니다.
3

🔌 USB 케이블로 PC 연결

전자칠판과 Windows PC를 USB 케이블로 연결합니다.

  • 전자칠판: USB-C 또는 Micro-USB 포트 (기기에 따라 다름)
  • PC: USB-A 포트에 연결
  • 데이터 전송을 지원하는 케이블인지 확인 (충전 전용 케이블 불가)
4

🔓 USB 디버깅 허용 팝업 승인

USB 케이블 연결 후 전자칠판 화면에 팝업이 표시됩니다.

🔐

[ "USB 디버깅을 허용하시겠습니까?" 팝업 ]

이 컴퓨터를 항상 허용 체크 → [허용] 터치

💡
"이 컴퓨터를 항상 허용" 체크박스에 체크하면 다음부터 팝업 없이 자동 연결됩니다.

4. ADB 설치 (PC에서)

ℹ️
ADB(Android Debug Bridge)는 PC에서 안드로이드 기기를 제어하기 위한 도구입니다. 최초 1회만 설치하면 됩니다.
1

📥 Platform Tools 다운로드

아래 공식 주소에서 Windows용 Platform Tools를 다운로드합니다.

# 공식 다운로드 URL
https://developer.android.com/tools/releases/platform-tools

페이지에서 "Windows용 SDK Platform Tools 다운로드" 링크를 클릭합니다.

2

📂 압축 해제 및 파일 배치

다운로드한 platform-tools_*.zip 파일의 압축을 풀고, 아래 두 가지 방법 중 하나를 선택합니다.

✅ 방법 A (권장): autostart 폴더에 복사

압축 해제된 폴더에서 adb.exe 파일을 찾아
autostart\ 폴더에 복사합니다. 설치 스크립트가 자동으로 인식합니다.

platform-tools› adb.exe→ 복사 → autostart\adb.exe
🔧 방법 B: 시스템 환경변수 PATH에 추가

platform-tools 폴더 전체 경로를 Windows 환경변수 PATH에 추가하면 어느 위치에서든 adb 명령어를 사용할 수 있습니다.

# 환경변수 설정 (예시 경로)
C:\Users\사용자명\Downloads\platform-tools
3

🔍 ADB 연결 확인

명령 프롬프트(cmd) 또는 PowerShell을 열고 아래 명령으로 연결을 확인합니다.

# ADB 버전 확인
adb version

# 연결된 기기 목록 확인
adb devices

# 정상 연결 시 출력 예시:
# List of devices attached
# ABCD1234    device
⚠️
unauthorized로 표시된다면 전자칠판 화면에서 "USB 디버깅 허용" 팝업을 확인하고 [허용]을 누르세요.

🚀
5. 설치 방법

✅
USB 연결과 ADB 설정이 완료된 후 PC에서 아래 단계를 진행합니다.
1

📁 설치 스크립트 실행

autostart 폴더에서 아래 파일을 더블클릭합니다.

📁 autostart› 🖱️ 안드로이드_설치.bat (더블클릭)
🛡️
Windows 보안 경고가 표시되면 "추가 정보" → "실행"을 클릭합니다.
2

🔢 반 번호 입력

스크립트 실행 후 명령 프롬프트 창에 해당 교실의 반 번호를 입력합니다.

# 입력 예시 (학년-반)
반 번호를 입력하세요: 1-1   # 1학년 1반
반 번호를 입력하세요: 2-5   # 2학년 5반
반 번호를 입력하세요: 3-10  # 3학년 10반
3

🤖 자동 설정 완료

스크립트가 아래 작업을 자동으로 수행합니다.

  • ✅ Chrome 브라우저에서 교실 알림 페이지 자동 실행
  • ✅ 화면 꺼짐 방지 (Stay Awake) 설정
  • ✅ 볼륨 최대로 설정 (미디어 볼륨)
  • ✅ 전체화면 (Immersive Mode) 적용
🎉
설치 완료 후 전자칠판 화면에 교실 알림 페이지가 자동으로 표시됩니다.

🔄
6. 부팅 시 자동 실행 설정

전자칠판을 재시작해도 자동으로 교실 알림 페이지가 열리도록 설정합니다. 아래 3가지 방법 중 가장 적합한 방법을 선택하세요.

★ 가장 권장

방법 1
전자칠판 자체 시작 앱 설정

  1. 전자칠판 설정(⚙️) 열기
  2. [앱] 또는 [시작 앱] 또는 [키오스크 모드] 메뉴 찾기
  3. Chrome 또는 교실알림 PWA를 시작 앱으로 지정
  4. 전자칠판 재시작으로 확인
💡
제조사마다 메뉴 이름이 다를 수 있습니다 (예: "부팅 앱", "Launcher", "자동 시작").
방법 2

방법 2
PWA 홈 앱으로 설정

  1. Chrome에서 교실 알림 페이지 열기
  2. Chrome 메뉴 (⋮) → [홈 화면에 추가]
  3. 설정 → 앱 → 기본 앱 → 홈 앱에서 교실알림 PWA 선택
  4. 홈 버튼 누르면 자동 실행
⚠️
모든 기기에서 지원되지 않을 수 있습니다.
방법 3

방법 3
네트워크 원격 실행 (무선 ADB)

  1. 전자칠판에서
    설정 → 개발자 옵션 → ADB over network 또는 무선 디버깅 켜기
  2. 전자칠판 IP 주소 확인
    (설정 → Wi-Fi → 연결된 네트워크 상세)
  3. PC 명령 프롬프트에서:
adb connect [IP주소]:5555
# 예: adb connect 192.168.1.50:5555

이후 USB 없이 안드로이드_설치.bat 실행 가능

7. 초기 설정 (전자칠판 화면에서)

1

🏫 반 선택

교실 알림 페이지가 열리면 반 선택 화면이 표시됩니다.
해당 교실의 반을 터치하여 선택합니다.

🏫

[ 반 선택 화면 ]

1학년 ▸ 1반, 2반, 3반... / 2학년 ▸ ... / 3학년 ▸ ...

2

👆 소리 활성화 (중요!)

"화면을 터치하여 시작" 안내 화면이 표시됩니다.
반드시 화면을 한 번 터치해야 합니다.

🔊
이 단계를 건너뛰면 알림 소리가 나지 않습니다!
브라우저 정책상 사용자 제스처 없이는 소리를 재생할 수 없습니다.

터치 후 자동으로 전체화면 모드로 전환됩니다.

3

⚙️ 추가 설정 (선택)

화면 좌측 상단 ⚙️ 설정 버튼을 터치하면 세부 설정을 변경할 수 있습니다.

  • 🔔 호출음: 알림 사운드 종류 선택
  • 🗣️ 음성(TTS): 음성 안내 켜기/끄기
  • 🔊 볼륨: 알림 음량 조절 (0~100%)
  • 🌙 야간 모드: 화면 밝기 자동 조절

9. 제거 방법

🔌 USB 연결 후 제거 스크립트 실행

전자칠판을 USB 케이블로 PC에 연결한 후, autostart 폴더에서 아래 파일을 더블클릭합니다.

📁 autostart› 🖱️ 안드로이드_제거.bat (더블클릭)

제거 스크립트는 ADB를 통해 설정된 내용을 초기화하고 Chrome 실행을 종료합니다.

ℹ️
제거 후 자동 실행 설정(시작 앱 지정 등)은 전자칠판 설정에서 직접 되돌려야 합니다.

10. 문제 해결

🔴 증상 ✅ 해결 방법
ADB 연결 안 됨
(unauthorized / offline)
  • 전자칠판에서 USB 디버깅 켜짐 여부 확인
  • USB 케이블을 데이터 전송 가능 케이블로 교체
  • 전자칠판 화면에서 "USB 디버깅 허용" 팝업 확인 후 [허용]
  • adb kill-server 후 재연결 시도
  • 기기 재부팅 후 재시도
소리 안 남
(알림은 오는데 음소거)
  • "화면을 터치하여 시작" 화면에서 반드시 터치하여 소리 활성화
  • 전자칠판 볼륨(하드웨어 버튼) 최대로 올리기
  • 설정(⚙️)에서 앱 볼륨 확인
  • Chrome 탭이 음소거 상태인지 확인 (주소창 옆 🔇 아이콘)
연결 끊김
(알림이 오지 않음)
  • Wi-Fi 연결 상태 확인 (설정 → Wi-Fi)
  • 화면을 터치하여 재연결 시도
  • 페이지 새로고침 (F5 또는 Chrome 메뉴)
  • 라우터/공유기 재시작
화면 꺼짐
(일정 시간 후 꺼짐)
  • 설정 › 디스플레이 › 자동 꺼짐(절전)을 최대값으로 설정
  • 개발자 옵션에서 "충전 중 화면 켜짐 유지" 활성화
  • 전원 연결(충전 중) 상태 유지
  • 설치 스크립트를 다시 실행하여 Wake Lock 재적용
Chrome 없음
(앱 설치 불가)
  • 기본 브라우저(WebView 기반)로 접속 시도
  • Google Play Store에서 Chrome 설치 (Play Store 없으면 APK 사이드로드)
  • Samsung Internet, Firefox 등 대체 브라우저 사용 (일부 기능 제한)
페이지 로드 안 됨
(흰 화면 또는 오류)
  • 인터넷 연결 확인 (Wi-Fi)
  • Chrome 앱 캐시 지우기: 설정 → 앱 → Chrome → 저장소 → 캐시 삭제
  • Chrome 업데이트 확인
  • 다른 기기에서 동일 URL 접속하여 서버 상태 확인
📞
그래도 해결이 안 된다면: 전자칠판을 재부팅 후 처음부터 다시 설치를 시도해보세요. 대부분의 문제는 재부팅 + 재설치로 해결됩니다.
